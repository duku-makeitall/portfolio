# [PRD] 소프트웨어 에듀테크 포트폴리오 웹사이트 요구사항 정의서

본 문서는 소프트웨어 에듀테크(Edutech) 기업의 채용 담당자를 타겟으로 하는 **이승화** 님의 개인 포트폴리오 웹사이트 구축을 위한 제품 요구사항 정의서(PRD)입니다. 웹 개발 초보자도 쉽게 이해하고 구현할 수 있도록 구체적이고 친절하게 작성되었습니다.

---

## 1. 프로젝트 개요 (Overview)

* **프로젝트명:** 이승화의 소프트웨어 에듀테크 포트폴리오 (Lee Seung-hwa's Edutech Portfolio)
* **목적:** 소프트웨어 에듀테크(Edutech) 기업의 교육콘텐츠 개발 직무 취업
* **주요 타겟:** 테크/에듀테크 기업의 채용 담당자, 인사담당자 및 기술 면접관
* **기획의 핵심 방향:** 
  1. **전문성 어필:** 아두이노 및 라즈베리파이 등 하드웨어(피지컬 컴퓨팅) 제어 역량과 교육 콘텐츠 기획 및 설계 역량 증명
  2. **에듀테크 최적화 UX:** 교육 서비스에 맞는 따뜻하고 친근하며 직관적인 디자인과 인터랙션 제공
  3. **쉬운 유지보수:** 복잡한 데이터베이스 구축 없이 단 한 개의 JSON 설정 파일 수정만으로 포트폴리오 정보(자기소개, 프로젝트 등)를 수정할 수 있는 정적 업데이트 구조 설계

---

## 2. 디자인 콘셉트 & UX (Design & Aesthetics)

에듀테크 기업의 감성에 부합하도록 **"친근하고, 활기차며, 신뢰감을 주는"** 톤앤매너로 디자인합니다.

* **컬러 팔레트 (Color Palette):**
  * **주 핵심 색상 (Primary):** 소프트웨어 교육의 활기참을 나타내는 옐로우/오렌지 계열 (`#FFB800` ~ `#FF8A00`)
  * **보조 색상 (Secondary):** 기술과 환경, 성장을 상징하는 산뜻한 그린/민트 계열 (`#2ECC71` ~ `#1ABC9C`)
  * **배경 색상 (Background):** 눈이 편안한 미색(Off-white) 및 매우 소프트한 그레이 테두리 (`#F9F9FB`, `#F0F2F5`)
  * **텍스트 색상 (Text):** 가독성이 높은 짙은 차콜 그레이 (`#2C3E50`)
* **타이포그래피 (Typography):**
  * 브라우저 기본 폰트 대신 구글 폰트의 **'Line Seed Sans KR'** 또는 **'Inter' + 'Pretendard'** 조합을 사용하여 트렌디하고 가독성 높은 폰트 제공
* **마이크로 애니메이션 (Micro-animations):**
  * 마우스 호버(Hover) 시 카드가 살짝 위로 떠오르는 효과 (Lift-up)
  * 버튼을 클릭할 때 통통 튀는 느낌을 주는 바운스 효과 (Scale down & up)
  * 스크롤을 내릴 때 각 섹션이 자연스럽게 위로 솟아오르며 페이드인(Fade-in) 되는 스크롤 애니메이션

## 3. 핵심 기능 정의 (Functional Requirements)

### 3.1. 메인 화면 및 자기소개 섹션 (Home & About)
* **목적:** 접속하자마자 이승화 님이 '어떤 가치를 만드는 교육콘텐츠 개발자'인지 명확하게 인지시킵니다.
* **주요 구성 요소:**
  * **프로필 헤드라인:** 한 줄 소개 (예: *"하드웨어로 배움의 즐거움을 설계하는 에듀테크 콘텐츠 개발자, 이승화입니다."*)
  * **자기소개 카드:** 핵심 역량 세 줄 요약, 관심 분야(소프트웨어 교육 콘텐츠 개발, 메이커 교육, 피지컬 컴퓨팅) 태그 표시.
  * **사용 기술 스택 배지 (Badge):** 아두이노(Arduino), 라즈베리파이(Raspberry Pi), Python, C/C++ 등을 시각적인 배지 형태로 나열.
* **[중요] 나만 편집 가능한 구조 (설정 파일 + 관리자 UI 방식):**
  * 웹사이트 소스코드 내부의 `profile-data.json` 파일에 자기소개 텍스트와 기술 스택 목록을 분리하여 저장합니다.
  * 이승화 님이 직접 JSON을 수정할 수도 있고, **로컬/배포 환경에서 관리자 페이지(login.html & admin.html)를 통해 GUI로 편리하게 내용을 편집**할 수도 있습니다. 편집 완료 후 JSON 파일로 내려받아 덮어쓰고 커밋/푸시(Commit & Push)하면 전체 사이트에 배포/업데이트됩니다.

  * 프로젝트 카드 리스트 (그리드 레이아웃)
  * 카드 클릭 시 프로젝트 상세 내용을 보여주는 모달 창(Modal Popup) 또는 전용 뷰어
  * 각 프로젝트별 태그 (예: `#아두이노`, `#라즈베리파이`, `#초등교육`, `#IoT`)
* **가상 프로젝트 구성 (에듀테크 최적화):**

| :--- | :--- | :--- | :--- |
| **Smart-Pet: 초등 피지컬 코딩 교구** | 아두이노, 서보 모터, 조도 센서, LED, C/C++ | 아이들이 조도 센서와 서보 터를 코딩하여 손을 가까이 대면 꼬리를 흔들며 반응하는 똑똑한 반려 로봇 제작 키트 및 4주차 교육 커리큘럼 설계 | - 하드웨어 회로 설계 능력<br>- 학습자 눈높이에 맞춘 스토리텔링형 교육 콘텐츠 설계 및 교안 기획력 |

### 3.3. 연락처 및 소셜 섹션 (Contact)
* **목적:** 채용 담당자가 이승화 님에게 간편하게 연락할 수 있는 통로를 제공합니다.
* **주요 구성 요소:**
  * **이메일 주소:** 클릭 시 사용자의 이메일 클라이언트가 열리는 링크 (`mailto:`) 및 주소 복사 버튼.
  * **깃허브 링크:** 이승화 님의 개발 코드를 볼 수 있는 깃허브 프로필로 이동하는 카드형 링크 (새 탭 열기).
  * **블로그/활동 기록 링크 (선택사항):** 교육 기획 관련 기록이나 메이커 활동을 정리한 노션/블로그 링크.

### 3.4. 관리자 로그인 및 설정 관리 (Admin Authentication & Management)
* **목적:** 개발 지식이 없거나 직접 JSON 텍스트를 수정하기 번거로울 때, 웹 UI를 통해 자기소개 정보 수정 및 프로젝트 추가/수정/삭제를 편리하게 수행할 수 있도록 합니다.
* **주요 구성 요소:**
  * **로그인 페이지 (`login.html`):**
    - 관리자 기능 진입 전 비밀번호를 확인하는 보안 단계입니다.
    - 정적 웹 호스팅 환경을 고려하여 복잡한 DB 인증 대신, 클라이언트 측에서 미리 정의된 비밀번호(또는 단방향 해시 비교)를 입력하여 세션을 검증하는 방식을 사용합니다.
    - 로그인 성공 시 브라우저 세션 상태(`sessionStorage`)에 로그인 성공 플래그를 저장하고 `admin.html`로 이동합니다.
    - 잘못된 비밀번호 입력 시 붉은색 안내 문구와 흔들림(Shake) 효과 같은 마이크로 인터랙션을 제공합니다.
  * **관리자 메인 페이지 (`admin.html`):**
    - 페이지 로드 시 로그인 여부(세션)를 체크하고, 인증되지 않은 사용자는 즉시 `login.html`로 강제 리다이렉트합니다.
    - **자기소개 편집 폼:** 프로필 헤드라인, 자기소개 설명글, 연락처(이메일, 깃허브, 블로그)를 입력 및 수정하는 텍스트 폼을 제공합니다. 기술 스택은 칩(Chip) 형태로 표시되며 자유롭게 추가/삭제할 수 있는 UI를 구성합니다.
    - **프로젝트 목록 & 추가/수정/삭제 폼:**
      - 현재 등록된 프로젝트 리스트를 테이블 또는 카드 뷰 형태로 노출합니다.
      - **추가/수정 폼 (또는 모달):** 프로젝트명, 사용 기술(태그), 한 줄 요약, 상세 설명, 이미지 경로, 깃허브 링크, 데모 링크를 입력할 수 있는 폼을 제공합니다.
      - **삭제 기능:** 특정 프로젝트 삭제 시 즉시 목록에서 제거되고 JSON 구조에 반영됩니다.
    - **데이터 임시 저장 & 로컬 미리보기 (LocalStorage):**
      - 관리자 페이지에서 변경 사항을 저장(Save)하면, 해당 데이터는 브라우저의 `localStorage`의 특정 키(예: `portfolio_draft_data`)에 즉시 저장됩니다.
      - 메인 페이지(`index.html`)가 로드될 때, `localStorage`에 임시 저장된 데이터가 존재하면 기본 `profile-data.json` 대신 해당 데이터를 우선적으로 바인딩하여, 개발 서버 재시작이나 깃 배포 없이도 즉시 화면 수정 결과를 완벽하게 미리보기할 수 있도록 지원합니다.
    - **최종 JSON 설정 파일 다운로드 (Export JSON):**
      - 수정 완료 후 "설정 파일 다운로드" 버튼을 클릭하면, 수정 사항이 모두 반영된 최종 JSON 데이터가 `profile-data.json`이라는 파일명으로 브라우저에서 다운로드됩니다.
      - 사용자는 다운로드한 파일을 프로젝트 폴더의 루트에 덮어쓴 후 깃 커밋 및 푸시하여 사이트에 영구 반영할 수 있습니다.
      - 초기화(Reset) 버튼을 통해 LocalStorage를 비우고 원본 JSON 상태로 언제든지 되돌릴 수 있는 기능을 제공합니다.

## 5. 비기능 요구사항 & 배포 계획

1. **검색엔진 최적화 (SEO):**
   * `<title>`: "이승화 \| 에듀테크 콘텐츠 개발자 포트폴리오"
   * `<meta name="description">`: "아두이노, 라즈베리파이 기반 소프트웨어 교육 콘텐츠를 설계하고 개발하는 이승화의 포트폴리오 사이트입니다."
   * 채용 담당자가 구글이나 네이버에 '이승화 포트폴리오', '에듀테크 이승화' 검색 시 노출될 수 있도록 메타 태그 완비.
2. **반응형 웹 디자인 (Responsive Design):**
   * 모바일(스마트폰), 태블릿, PC 모니터 화면 크기에 맞추어 레이아웃이 유연하게 변화하도록 구현 (특히 채용 담당자가 모바일로 링크를 확인하는 경우 대비).
3. **무료 호스팅 및 간편 배포:**
   * **GitHub Pages**를 활용하여 별도의 호스팅 비용 없이 평생 무료로 웹사이트를 배포 및 운영할 수 있는 환경 권장.
